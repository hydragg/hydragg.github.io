將使用vue cli的專案部署在github page上

6月 26, 2018

前言

        vue cli的好用大家都知道，但是想要部署將build完的作品放在github page上面時，會發現dist/index.html裡面資源引入的路徑都是 / 這時候就會有好幾種方法。

  1. 將dist/index.html 引入資源 / 改成 ./
  2. 只上傳dist底下的檔案
  3. 從webpack config設定

第一種只要每次build的就要重新修改一次，有點麻煩，第二種則是不能將整個專案傳上去，這樣協作或是在不同地方想拉下來做就很不方便，所以我選擇從webpack config設定，雖然vue cli已經出到3.0版本了，但大多數人應該還在使用2.0版本，這邊就一起說明了！

vue cli 2.0

<project name>/config/index.js

改成這樣之後，只要build完之後的index.html就會將 資源引入改為 ./

ps: vue cli template為webpack

vue cli 3.0

因為vue cli 3.0之後，如果要更改webpack的設定就要在根目錄下建立一個vue.config.js來調整webpack設置，更多內容可以查閱官方文件

<project name>/vue.config.js

這樣就大功告成了，這樣只要到<github name>.github.io/<project name>/dist/index.html看看有沒有成功

ps: 記得將.gitignore檔案把/dist 給移除掉，不然git會忽略dist
